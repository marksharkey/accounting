## The 5 Exact QBO Files You Need to Export

Before running the import, export these 5 files from QuickBooks Online and place them in `~/accounting/backend/`:

### 1. **Customers.csv**
- **Where:** Sales → Customers → Export (as XLS, then convert to CSV)
- **What:** All customers (company names, emails, addresses, open balances)
- **How to convert:** `libreoffice --headless --convert-to csv Customers.xls`

### 2. **ProductServiceList__*.csv**
- **Where:** Settings → Products & Services → Export (as CSV)
- **What:** Service catalog (~40 items)
- **Note:** Filename includes timestamp (auto-generated by QBO)

### 3. **PrecisionPros_Network_Sales_by_Product_Service_Detail.csv**
- **Where:** Reports → Sales by Product/Service Detail (Jan 1, 2023 to Today)
- **What:** All invoices and their line items (4,000+ invoices)
- **Used by:** Invoice import + Billing schedule import

### 4. **PrecisionPros_Network_A_R_Aging_Detail_Report.csv**
- **Where:** Reports → A/R Aging Detail (As of Today)
- **What:** Invoice aging data (which invoices are paid vs open)
- **Used by:** Invoice import (sets paid/open status)

### 5. **PrecisionPros_Network_Transaction_Detail_by_Account.csv**
- **Where:** Reports → Transaction Detail by Account (Jan 1, 2023 to Today, **Cash basis**)
- **What:** All expense transactions (~880 items)
- **Used by:** Expense import

**Time to export all files:** ~15 minutes  
**See:** `QBO_EXPORT_GUIDE.md` for detailed step-by-step instructions

## The Import Scripts & Tools

### 🚀 Master Import Runner: `run_full_import.py`
**The main command you'll use.**

```bash
# Preview (no database changes)
python3 run_full_import.py --dry-run

# Execute (commits changes)
python3 run_full_import.py --commit
```

**What it does:**
- ✓ Validates all CSV files exist
- ✓ Resets database (truncates data, preserves schema)
- ✓ Runs: services import → customers import → invoices import → expenses import → billing schedules import
- ✓ Runs: cleanup script (fixes unit prices, subtotals)
- ✓ Applies: manual fixes (e.g., invoice #48958)
- ✓ Runs: validation script
- ✓ Generates: comprehensive log file

**Time:** 5-10 minutes

---

### 📊 Validation Script: `validate_import.py`
**Compares QBO source to database to identify issues.**

```bash
python3 validate_import.py
```

**Reports:**
- Missing invoices (in QBO but not imported)
- Invoice date mismatches
- Line item integrity issues
- Subtotal discrepancies
- Customer count validation
- Actionable recommendations for fixes

## How to Use (Complete Workflow)

### Step 1: Export Data from QBO (15 minutes)
Follow the detailed instructions in `QBO_EXPORT_GUIDE.md`:
1. Customer export (convert `.xls` to `.csv`)
2. Product/Service export
3. Sales by Product/Service Detail report
4. A/R Aging Detail report
5. Transaction Detail by Account report (Cash basis)

Place all 5 files in: `~/accounting/backend/`

### Step 2: Preview Import (No Changes)
```bash
cd ~/accounting/backend
python3 run_full_import.py --dry-run
```

Review the output to verify all CSV files are found and the process looks correct.

### Step 3: Execute Import
```bash
python3 run_full_import.py --commit
```

The script will:
- Reset the database
- Import all data in sequence
- Run cleanup and manual fixes
- Validate results
- Generate log file with timestamp

### Step 4: Review Validation Report
Check the output for any issues:
- Missing invoices?
- Date mismatches?
- Line item problems?

If issues found, review `IMPORT_QUICK_START.md` troubleshooting section.

### Step 5: Manual Cleanup (If Needed)
The import sets defaults for all clients (autocc_recurring, late_fee_type, etc.). Review and adjust as needed.

**Total time:** ~20 minutes (15 export + 5 import)

## Known Issues & Workarounds

### Invoice #48958
- **Issue:** QBO exported $1.00 instead of $25.00
- **Workaround:** Automatically fixed by master runner

### 8 Customers Missing Email
- **Issue:** These customers exist but have no email in QBO
- **Workaround:** Manually add after import (list provided in IMPORT_PROCEDURES.md)

### 54 Customer Name Mismatches
- **Issue:** QBO invoices use "Name" field, customers export uses "Company name"
- **Workaround:** NAME_MAP in import scripts handles this automatically

### Missing Invoices
- **Issue:** Some invoices in QBO don't import
- **Cause:** Usually deleted customers or pre-2023 history
- **Solution:** Review validation report, check QBO manually

## Important Notes

⚠️ **Database Reset Warning**
- The master runner truncates all transaction data
- Does NOT truncate users, chart_of_accounts, company_info
- Always run `--dry-run` first if unsure

⚠️ **File Format Requirements**
- Customers must be `.csv` (convert from `.xls` using libreoffice)
- All other files must be CSV format from QBO
- See `QBO_EXPORT_GUIDE.md` for exact export paths

⚠️ **Date Ranges**
- Sales reports: Jan 1, 2023 to Today (not earlier)
- A/R Aging: As of Today (current date)
- This ensures consistency across all imports
